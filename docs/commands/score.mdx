title: "score"

```bash
skillrank score <skill> [options]
```

Same pipeline as [`bench`](/commands/bench), but for one skill only. Faster, cheaper, useful for iterating on a skill you're developing.

<Warning>
  Requires `ANTHROPIC_API_KEY`. See [Installation](/installation#set-up-your-api-key).
</Warning>

## Usage

```bash
skillrank score frontend-design
```

## When to use score vs bench

| Use case | Command |
|---|---|
| Iterating on a skill you're writing | `score` |
| Checking if a single skill is worth keeping | `score` |
| Auditing all installed skills | `bench` |
| Generating a shareable report | `bench --md` |

## Tip: iterating on skill descriptions

The skill description drives task generation, which drives scoring. If your skill has a vague description, the generated tasks will be vague too.

Workflow:

```bash
# 1. Write your skill
vim ~/.claude/skills/my-skill/SKILL.md

# 2. Score it
skillrank score my-skill

# 3. Rewrite the description based on what tasks were generated

# 4. Re-score
skillrank score my-skill
```

Most skills go from a score in the 40s to 80+ after one or two description rewrites.
