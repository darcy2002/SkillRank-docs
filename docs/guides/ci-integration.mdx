description: "Bench your skills automatically on every PR. Catch broken skills before they ship."
---

If you maintain a stack or write custom skills, integrate skillrank into your CI to catch regressions.

## GitHub Actions

```yaml
# .github/workflows/bench.yml
name: Bench Skills

on:
  pull_request:
    paths:
      - 'skills/**/SKILL.md'
      - 'skillrank.yaml'
  push:
    branches: [main]

jobs:
  bench:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      
      - name: Install skillrank
        run: npm install -g skillrank
      
      - name: Install stack
        run: skillrank install -f ./skillrank.yaml -y
      
      - name: Run bench
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: skillrank bench --json --md
      
      - name: Upload bench report
        uses: actions/upload-artifact@v4
        with:
          name: bench-report
          path: |
            bench-report.json
            bench-report.md
      
      - name: Comment on PR
        if: github.event_name == 'pull_request'
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const report = fs.readFileSync('bench-report.md', 'utf8');
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: report
            });
```

## Required setup

<Steps>
  <Step title="Add the secret">
    Go to Repo Settings → Secrets and variables → Actions → New repository secret.
    Name: `ANTHROPIC_API_KEY`. Value: your Anthropic API key.
  </Step>
  <Step title="Add the workflow">
    Create `.github/workflows/bench.yml` with the YAML above.
  </Step>
  <Step title="Commit and push">
    Open a PR that changes a SKILL.md you'll see the bench report comment automatically.
  </Step>
</Steps>

## Fail the build on regressions

Want to block PRs that introduce broken skills? Add a check step:

```yaml
- name: Fail on broken skills
  run: |
    BROKEN=$(jq '[.skills[] | select(.status == "broken")] | length' bench-report.json)
    if [ "$BROKEN" -gt 0 ]; then
      echo "❌ $BROKEN broken skill(s) detected. Blocking merge."
      exit 1
    fi
```

## GitLab CI

```yaml
# .gitlab-ci.yml
bench:
  image: node:20
  script:
    - npm install -g skillrank
    - skillrank install -f ./skillrank.yaml -y
    - skillrank bench --json --md
  artifacts:
    paths:
      - bench-report.json
      - bench-report.md
  only:
    - merge_requests
```

## Cost considerations

CI bench runs add up. To keep costs in check:

- **Bench only on PRs that touch skills** use `paths` filters
- **Use Haiku for grading in CI** `--grader-model claude-haiku-4-5-20251001`
- **Reduce tasks in CI** `--tasks 1` for fast feedback, full `--tasks 3` on main
- **Don't bench on every commit** limit to PR events

A typical stack with 10 skills running on each PR costs:

- With Haiku grading + 1 task: ~$0.04 per run
- With Sonnet defaults + 3 tasks: ~$0.20 per run

## Scheduled audits

Re-bench your stack quarterly to catch skill rot (when underlying tools change but your skills don't):

```yaml
on:
  schedule:
    - cron: '0 0 1 */3 *'  # Quarterly on the 1st
```

If scores drop, open an issue automatically:

```yaml
- name: Open issue on regression
  if: failure()
  uses: actions/github-script@v7
  with:
    script: |
      github.rest.issues.create({
        owner: context.repo.owner,
        repo: context.repo.repo,
        title: 'Quarterly bench: scores regressed',
        body: 'Bench scores dropped. See latest run for details.'
      });
```
